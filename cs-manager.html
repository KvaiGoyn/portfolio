CS Server Manager

Настольный пульт над штатными dedicated-серверами CS2 клуба: создание и запуск серверов, управление игроками по RCON, живые логи консоли, установка плагинов и Metamod — всё из одного окна, без ручного правления конфигов и командной строки.

[ Задача ]

Зачем это нужно

В клубе крутится несколько CS2-серверов под тренировки, турниры и обычные катки. Их приходится поднимать, гасить и перенастраивать вручную — через командную строку, правку конфигов и RCON-команды на память. Это медленно и требует человека, который помнит все аргументы запуска.

Задача — свести всё управление в одно приложение. Чтобы администратор смены запускал сервер кнопкой, видел кто на нём играет, кикал нарушителей, читал логи и ставил плагины без терминала и без риска сломать рабочий конфиг.

Осознанное решение: не оборачивать серверы в Docker, а строить надстройку над штатным dedicated-сервером Valve. Так проще обновлять игру и разбираться, когда что-то ломается на живом турнире.

[ Инженерный разбор ]

Чем задача была сложна

Это не обёртка над готовым API, а прямое управление процессами игрового сервера, сетевым протоколом и файловой системой плагинов.

[ 01 ]

Своя реализация RCON поверх TCP

Управление сервером идёт по протоколу RCON: установить TCP-соединение, аутентифицироваться, слать команды и разбирать бинарные пакеты ответа — статус, список игроков, результат кика.

Как решеноTCP-клиент с парсингом RCON-пакетов, командами статуса и кика игроков и ботов прямо из интерфейса.
[ 02 ]

Запуск и контроль процесса cs2

Сервер поднимается через spawn исполняемого файла с набором аргументов, а его жизнь надо отслеживать: жив ли процесс, не упал ли. Кроссплатформенно — через tasklist и kill -0.

Как решеноМониторинг процессов с опросом каждые 5 секунд и авто-обновлением статуса каждой карточки сервера.
[ 03 ]

Чтение консоли с диска в реальном времени

Консоль сервера пишется в console.log на диске. Нужно читать файл по мере роста, не перечитывая целиком, и раскрашивать вывод для читаемости.

Как решеноЧтение лога раз в секунду с цветовой разметкой строк — консоль сервера видна прямо в приложении.
[ 04 ]

Установка и переключение плагинов

Плагины включаются и выключаются переименованием папок и деплоем в counterstrikesharp/plugins/. Metamod ставится скачиванием ZIP с GitHub, распаковкой и отслеживанием прогресса.

Как решеноУправление плагинами через файловую систему + загрузчик Metamod с распаковкой и прогресс-баром.
[ 05 ]

Frameless-окно и тёмная тема

Стандартная рамка окна ломает цельный вид. Frameless-окно требует своей реализации перетаскивания, кнопок свернуть/развернуть/закрыть и 800+ строк кастомного CSS.

Как решеноСобственный тайтлбар, тёмная тема и модальная система вместо нативных браузерных alert().
[ 06 ]

Конфиги серверов между запусками

Настройки серверов, пути и параметры должны переживать перезапуск приложения без развёртывания базы данных.

Как решеноJSON-конфиги в ~/.cssm/ — просто, переносимо, легко бэкапить.

На чём собрано

Оболочка
Electron. Десктоп-приложение с доступом к процессам, файловой системе и сети.
Фронтенд
Vue 3 + Ant Design Vue 4. Компонентный интерфейс, поверх — 800+ строк кастомного CSS в тёмной теме.
Управление
RCON поверх TCP. Аутентификация, команды, парсинг пакетов, статус и кик игроков.
Процессы
Node child_process. spawn cs2, мониторинг через tasklist / kill -0, опрос каждые 5 с.
Графики
ECharts. Статистика серверов в виде живых графиков.
Хранение
JSON-конфиги в ~/.cssm/ — без базы данных, переносимо.
