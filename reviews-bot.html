Reviews Aggregator

Сервис собирает отзывы о заведении сразу с трёх карт-агрегаторов — Яндекс.Карты, 2ГИС и Google Business Profile — и присылает новые в Telegram команде. Вместо ручного обхода трёх сайтов репутация приходит сама, а новый источник добавляется одним классом.

Три источника, у каждого свой характер

Собрать отзывы с одной площадки — вечер работы. Собрать с трёх разных, привести к единому виду и ни разу не задублировать — принципиально другая задача.

[ 01 ]

Три площадки — три способа достать данные

Яндекс, 2ГИС и Google отдают отзывы совершенно по-разному: где-то API с ключом, где-то — OAuth2 с refresh-токеном. Каждый источник живёт по своим правилам, а на выходе формат сообщения должен быть один.

Как решено Общий интерфейс BaseParser.parse() и единый DTO ReviewDTO: новый источник — это новый класс-парсер, ядро и рассылка не меняются.
[ 02 ]

Google Business Profile через OAuth2

Google не отдаёт отзывы по простому ключу — нужен полноценный OAuth2 с получением refresh-токена, аккаунтом и локацией. Самая тяжёлая из трёх интеграций.

Как решено Отдельный скрипт получения refresh-токена; источник включается опционально — только когда заданы креды, иначе работают Яндекс и 2ГИС.
[ 03 ]

Один отзыв — одно уведомление

При опросе раз в 30 минут те же отзывы возвращаются снова и снова. Нужно надёжно отличить «уже видели» от «новое» — даже когда у отзыва нет стабильного ID.

Как решено Fingerprint — либо внешний ID отзыва, либо md5 от «автор:текст». Состояние хранится между запусками, так что перезапуск не превращается в шквал дублей.
[ 04 ]

Работать самому, без присмотра

Бот должен опрашивать источники по расписанию в фоне, параллельно отвечая на команды, и переживать перезапуски без потери состояния.

Как решено asyncio-планировщик стартует в фоне при запуске бота; состояние в JSON, деплой через Docker, доступ к боту ограничен списком чатов.
[ Экраны · 01–03 ]

Отзывы приходят в Telegram

Три источника — один формат сообщения. Плейсхолдеры заменяются автоматически при добавлении файлов.

Как добавить

Файлы PNG в assets/reviews/ с именами из плейсхолдеров — вертикальный формат под скрины телефона.

[ Архитектура ]

Как это устроено

Ядро
aiogram + asyncio. Telegram-бот с фоновым планировщиком, который стартует при запуске и опрашивает источники по расписанию.
Парсеры
Отдельный класс на источник (Яндекс, 2ГИС, Google) поверх общего интерфейса BaseParser.
Модель данных
Единый ReviewDTO — отзывы из всех источников приводятся к одному виду для рассылки.
Google API
OAuth2 Business Profile API плюс отдельный скрипт получения refresh-токена.
Дедупликация
Fingerprint по external_id или md5(автор:текст); состояние в state.json.
Деплой
Docker + docker-compose, доступ к боту ограничен списком чатов.
В планах

Отправка тех же уведомлений в Bitrix24 — переиспользуя паттерн вебхук-интеграции из отсека Booking Notifier.
